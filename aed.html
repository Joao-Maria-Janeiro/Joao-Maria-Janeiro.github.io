Explaining the project

The project is made up of six challenges. All the challenges consist of finding paths in a map (which is represented by a matrix witha cost at each cell). It's important to note that we can only move in adjacent steps so 8 options per cell (up, down, left, right and the corners).

In order to simplify the data manipulation I created a Map structure (definition up next) which stores all the data given in the input (besides the actual map).

The challenges

Variant A

Given a starting point, x0, find a path with exactly k steps so that the cells are visited in ascending order of their values, or indicate the path does not exist.

How did I go about solving this?

To approach this problem it was obvious that we had to use a graph search algorithm. Now all that was left was to figure out if the best one was DFS or BFS. Evaluating the given description for the variant I thought that if I applied a BFS it would take a lot longer to find what we want because we would do a lot of unecessary work and try to see too many paths at once while if we used the DFS we would check path by path checking if each cell has a bigger value than the previous one and, if so, we would keep on this path until one of two options, we found a path with k steps (in this case the path was found and we only had to save it and exit the fucntion) or this cell has a smaller value then the previous one which in this case we would discard this cell and try the next one.

In order to actually solve this problem I had to choose a stoping condition for the recursion. So the logical one was when the number of steps was equal to k, if that happens, we set the "found" flag to 1 so we can then store the path in the array. In order to get the adjacencies of each cell I simply call the get_adjacent function which return an array of type "Point" (definition up next) with those adjacencies.

Then we iterate throufh all the adjacencies of the cuurent cell and check if an adjacency is bigger than the current cell, if so we go to it and check its adjacencies and so on, creating a path.

Without the break in the if statement the code was writing a lot of unnecessary work so after some thought I realised that we had to break on each point because we don't to iterate to any more adjacencies we just want the points we passed by on the path.
